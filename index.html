Machine Learning · Classificação + Regressão
Aprovação de Crédito com ML
PythonScikit-learnRandomForestRegressão LinearPandasMatplotlib
Sobre o projeto

Automatiza decisões de concessão de empréstimos bancários e calcula a taxa de juros ideal para cada perfil de cliente. Combina um classificador de aprovação com um modelo de regressão para precificação de risco.

Decisões de crédito mal calibradas custam bilhões anualmente em inadimplência ou em clientes bons que são negados injustamente. Este projeto substitui intuição por predição quantitativa — dois modelos em pipeline: um classificador para aprovar ou negar, e um regressor para calcular a taxa proporcional ao risco real de cada perfil.

Resultado prático
📌 Impacto

Reduz risco de inadimplência e acelera aprovações. Substitui intuição por decisão baseada em dados, com transparência total sobre os fatores de cada decisão. Bancos e fintechs automatizam análises que dependem de analistas humanos. Cooperativas oferecem taxas personalizadas e justas por perfil. Empresas de empréstimo escalam operações com decisões consistentes e auditáveis.

Como foi feito
Python + Scikit-learn
Biblioteca principal de Machine Learning — usada para treinar, avaliar e comparar os modelos de classificação e regressão.
Random Forest
Algoritmo baseado em múltiplas árvores de decisão (regras do tipo "se renda > X e dívida = 0, então aprovar"). Combinar várias árvores aumenta a precisão e reduz erros de generalização.
FastAPI
API REST para servir o modelo em produção — recebe os dados do cliente e retorna a decisão de aprovação + taxa calculada em tempo real.
Pandas
Manipulação e limpeza de dados em tabelas — equivalente a um Excel programável e muito mais poderoso.
Matplotlib
Biblioteca de visualização — usada para gerar gráficos de análise e interpretação dos resultados do modelo.

Predição de Churn Bancário 📓 Notebook 🚀 Testar o modelo
Machine Learning · Classificação
Predição de Churn Bancário
SklearnMLflowSMOTEGradientBoostingRandomForestSeaborn
Sobre o projeto

Identifica clientes com alta probabilidade de cancelamento antes que saiam, permitindo ação proativa de retenção. Avalia e compara três modelos com rastreamento automático via MLflow.

Adquirir um novo cliente custa 5–7× mais do que reter um existente. Este modelo detecta sinais de saída antes do cancelamento — probabilidade de churn calculada por perfil, com recomendações de ação para cada nível de risco. O desafio principal é o desequilíbrio nos dados — clientes que cancelam são sempre minoria. Para resolver isso, foi usado SMOTE combinado com rastreamento de experimentos via MLflow.

Resultado prático
📌 Impacto

Ação preventiva antes da saída do cliente. Prioriza esforços de retenção nos perfis de maior risco, com decisões consistentes e auditáveis. Bancos e fintechs acionam equipes de retenção antes do cancelamento. Serviços de assinatura reduzem cancelamentos com ofertas personalizadas. Times comerciais otimizam seu esforço focando nos perfis de maior risco.

Como foi feito
Python + Scikit-learn
Gradient Boosting e Random Forest treinados e comparados na mesma pipeline de avaliação.
MLflow
Plataforma de rastreamento de experimentos — registra automaticamente métricas, parâmetros e resultados de cada treino, facilitando comparações entre modelos.
SMOTE
Técnica de balanceamento de dados — gera exemplos sintéticos da classe minoritária para o modelo aprender a identificar cancelamentos com mais precisão.
Seaborn
Biblioteca de visualização de dados — usada na análise exploratória para entender o perfil dos clientes que cancelam.
FastAPI + Render
Deploy do modelo como API REST, servindo previsões em tempo real com interface interativa.

Análise de Reviews com NLP + LLM
spaCyTF-IDFWord2VecQwen 0.5BPrompt Eng.Python
Sobre o projeto

Transforma milhares de avaliações de clientes em inteligência de produto automaticamente. Analisa sentimento, extrai palavras-chave e gera resumos executivos por produto usando NLP clássico e modelos de linguagem.

Empresas com catálogos de centenas de produtos recebem milhares de reviews diariamente — ler manualmente é inviável. Este pipeline transforma texto bruto em inteligência estruturada: sentimento por produto, tendências emergentes e resumos executivos gerados por LLM, prontos para decisão sem intervenção humana.

Resultado prático
📌 Impacto

Elimina análise manual de reviews. Entrega sentimento, tendências e recomendações prontas para decisão. E-commerce monitora reputação de produtos em tempo real. Times de produto identificam o que clientes valorizam e o que gera reclamação. Marketing extrai atributos associados a avaliações positivas para usar em campanhas.

Como foi feito
spaCy
Biblioteca de processamento de linguagem natural — usada para limpeza, tokenização (divisão do texto em palavras) e extração de entidades relevantes.
TF-IDF
Técnica que mede a importância de uma palavra dentro de um documento comparado ao conjunto inteiro — identifica o que é realmente relevante em cada avaliação.
Word2Vec
Modelo que aprende associações entre palavras — permite entender que "gostoso" e "saboroso" têm significados parecidos sem ser programado para isso.
Qwen 0.5B + Prompt Eng.
Modelo de linguagem para gerar os resumos executivos. O prompt engineering garante que a saída seja estruturada e relevante para negócios.

Dashboard de Vendas — Power BI 🚀 Ver dashboard
Business Intelligence · Data Visualization
Dashboard de Vendas — Power BI
Power BIDAXPower QueryExcelData Viz
Sobre o projeto

Dashboard interativo de vendas com visão geral por produto, região e marca. Consolida dados de 247 mil transações com análise de faturamento, receita, gastos e lucro em painel dinâmico filtrado por período.

Relatórios manuais em Excel são lentos, propensos a erro e não respondem perguntas de negócio em tempo real. Este dashboard transforma 247 mil registros em visão executiva interativa — qualquer stakeholder filtra por período, região ou categoria e obtém a resposta sem depender de um analista.

Resultado prático
📌 Impacto

Visão executiva em tempo real — decisões de produto, região e mix de marcas com base em dados. Gestão comercial acompanha faturamento e margem por período sem relatórios manuais. Times de produto identificam quais linhas e marcas puxam volume. Alta gestão tem painel consolidado para decisões estratégicas com drill-down completo.

Como foi feito
Power BI
Plataforma de Business Intelligence para criação de dashboards interativos e visuais dinâmicos conectados aos dados.
DAX
Linguagem de fórmulas do Power BI — usada para criar medidas como lucro %, ticket médio e receita líquida que respondem dinamicamente aos filtros.
Power Query
Motor de ETL integrado ao Power BI — automatiza a limpeza, transformação e integração dos dados brutos sem código manual.
Excel
Utilizado na etapa inicial de preparação dos dados — limpeza de inconsistências e padronização antes da ingestão no Power BI.
