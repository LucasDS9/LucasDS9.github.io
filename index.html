Como foi feito

O projeto usa dois modelos juntos. O primeiro é um classificador (modelo de decisão binária — aprova ou nega) que analisa o perfil do cliente: score de crédito, renda comprometida e dívidas ativas. O segundo é um modelo de regressão (previsão numérica) que calcula a taxa de juros adequada ao risco de cada perfil aprovado.

Antes de treinar, os dados passaram por limpeza e engenharia de variáveis (criação de indicadores novos, como o percentual da renda comprometida) que o modelo aprende a usar como sinal de risco.

Ferramentas utilizadas
Scikit-learn
Biblioteca principal de Machine Learning em Python — usada para treinar, avaliar e comparar os modelos de classificação e regressão.
Random Forest
Algoritmo baseado em múltiplas árvores de decisão (regras do tipo "se renda > X e dívida = 0, então aprovar"). Combinar várias árvores aumenta a precisão e reduz erros.
Pandas
Ferramenta para manipulação e limpeza de dados em tabelas — equivalente a um Excel programável e muito mais poderoso.
Matplotlib
Biblioteca de visualização — usada para gerar gráficos de análise e interpretação dos resultados do modelo.

Como foi feito

O desafio principal é o desequilíbrio nos dados — clientes que cancelam são sempre minoria. Para resolver isso, foi usado SMOTE (técnica que cria exemplos sintéticos da classe minoritária, equilibrando o aprendizado do modelo). Três algoritmos foram treinados e comparados com rastreamento automático via MLflow (plataforma que registra os experimentos de ML como um diário de treinos).

O modelo final foi escolhido com base no recall (capacidade de não deixar escapar nenhum cliente em risco real) — a métrica mais importante para um problema de retenção.

Ferramentas utilizadas
Scikit-learn
Biblioteca principal para treinar e avaliar os modelos de classificação — Gradient Boosting e Random Forest.
SMOTE
Técnica de balanceamento de dados — gera exemplos sintéticos (artificiais) da classe minoritária para o modelo aprender a identificar cancelamentos com mais precisão.
MLflow
Plataforma de rastreamento de experimentos — registra automaticamente métricas, parâmetros e resultados de cada treino, facilitando comparações entre modelos.
Seaborn
Biblioteca de visualização de dados — usada na análise exploratória para entender o perfil dos clientes que cancelam.

Como foi feito

O pipeline começa com limpeza dos textos — remoção de pontuações e palavras sem significado (como "de", "o", "a"). Em seguida aplica-se análise de sentimento (classificação automática do tom de cada avaliação: positivo, neutro ou negativo).

Para identificar os temas relevantes usa-se TF-IDF (pontuação que destaca palavras importantes dentro de um conjunto de textos) e Word2Vec (modelo que aprende o significado de palavras pelo contexto em que aparecem). Por fim, um modelo de linguagem gera resumos executivos prontos para leitura.

Ferramentas utilizadas
spaCy
Biblioteca de processamento de linguagem natural — usada para limpeza, tokenização (divisão do texto em palavras) e extração de entidades relevantes.
TF-IDF
Técnica que mede a importância de uma palavra dentro de um documento comparado ao conjunto inteiro — identifica o que é realmente relevante em cada avaliação.
Word2Vec
Modelo que aprende associações entre palavras — permite entender que "gostoso" e "saboroso" têm significados parecidos sem ser programado para isso.
Qwen 0.5B + Prompt Engineering
Modelo de linguagem para gerar os resumos executivos. O prompt engineering (escrita de instruções precisas para o modelo) garante que a saída seja estruturada e relevante para negócios.

Como foi feito

Os dados brutos foram extraídos e tratados no Excel com limpeza de inconsistências e padronização de colunas. Em seguida, o Power Query automatizou a transformação e a integração das fontes, criando um pipeline de atualização sem intervenção manual.

As métricas de negócio — lucro percentual, ticket médio e receita líquida — foram calculadas com funções DAX (Data Analysis Expressions), que permitem agregações dinâmicas respondendo aos filtros do painel. O resultado é um dashboard totalmente interativo com slicers de ano, mês, região e tipo de produto.

Ferramentas utilizadas
Power BI
Plataforma de Business Intelligence para criação de dashboards interativos e visuais dinâmicos conectados aos dados.
DAX (Data Analysis Expressions)
Linguagem de fórmulas do Power BI — usada para criar medidas como lucro %, ticket médio e receita líquida que respondem dinamicamente aos filtros.
Power Query
Motor de ETL integrado ao Power BI — automatiza a limpeza, transformação e integração dos dados brutos sem código manual.
Excel
Utilizado na etapa inicial de preparação dos dados — limpeza de inconsistências e padronização antes da ingestão no Power BI.
