Sistema RAG Q&A (PT)
Mini RAG PRO – Perguntas e Respostas com RAG em português
PythonTransformersChromaDBLangChainQwen2.5-1.5BGradio

Sistema completo de Retrieval-Augmented Generation (RAG) desenvolvido em Python que permite realizar perguntas sobre documentos PDF utilizando embeddings semânticos, banco vetorial persistente, re-ranking neural e LLMs para geração de respostas contextualizadas em português.

O projeto implementa uma arquitetura moderna de IA generativa semelhante às utilizadas em aplicações reais como assistentes corporativos, sistemas jurídicos e chatbots especializados. Cobre todo o ciclo de vida RAG: ingestão de documentos, chunking adaptativo, geração de embeddings, indexação vetorial, recuperação semântica, re-ranking e avaliação quantitativa.

A arquitetura integra ChromaDB como banco vetorial persistente, LangChain Text Splitters para chunking adaptativo com granularidade variável, e Transformers para execução do modelo generativo Qwen2.5-1.5B. O sistema inclui ainda um módulo de avaliação automática LLM-as-Judge com métricas de relevância, coerência, correção e completude, complementado por similaridade de embeddings para fidelidade e concisão.

ML end-to-end para decisões bancárias de empréstimo
PythonScikit-learnRandomForestRegressão LinearPandasNumpyMatplotlib

Projeto completo de Machine Learning focado em prever aprovação de empréstimos bancários. Além da classificação binária, o projeto realiza regressão para estimar a taxa de juros ideal para cada cliente aprovado — uma solução end-to-end para decisões de crédito.

Combina EDA aprofundada com insights estratégicos, regras de negócio customizadas, RandomForestClassifier para aprovação e regressão linear para precificação de taxas. O notebook inclui análise detalhada dos fatores que mais impactam a concessão de crédito.

A solução auxilia instituições financeiras a reduzir riscos, automatizar processos e otimizar taxas oferecidas, sendo facilmente expansível para cenários reais de produção.

Análise de Produtos Amazon (NLP)
Pipeline NLP + LLM sobre o dataset Amazon Fine Foods
spaCyTF-IDFWord2VecQwen 0.5BPrompt EngineeringPython

Pipeline completo de Processamento de Linguagem Natural (NLP) voltado à extração de insights estruturados a partir de avaliações textuais do Amazon Fine Foods dataset. O objetivo é transformar reviews não estruturadas em informações estratégicas por produto, combinando técnicas clássicas de NLP com análise via LLMs.

O pipeline cobre todas as etapas: ingestão dos dados, limpeza textual e feature engineering, vetorização via TF-IDF e Word2Vec, agregação de sentimento por produto e geração de métricas estruturadas. O modelo Qwen 0.5B é utilizado com prompt engineering para gerar insights executivos, analisar críticas e recomendar melhorias por produto.

Em ambiente corporativo, pode ser aplicado para monitoramento de produtos, análise da experiência do cliente, apoio à tomada de decisão e escalabilidade analítica — substituindo análises manuais por um pipeline automatizado de NLP + LLM.

Classificador de Churn Bancário
Predição de cancelamento com MLflow tracking e SMOTE
PythonScikit-learnMLflowSMOTEGradientBoostingRandomForestPandasSeaborn

Projeto completo de Machine Learning focado em prever quais clientes bancários têm maior probabilidade de cancelar seus serviços (churn). Além da predição binária, o projeto identifica os principais fatores que influenciam o cancelamento, entregando insights estratégicos para retenção de clientes.

O pipeline utiliza SMOTE para balanceamento de classes, encapsulado em um pipeline scikit-learn com pré-processamento via ColumnTransformer e OrdinalEncoder. Três modelos foram avaliados — Logistic Regression, Random Forest e Gradient Boosting — todos rastreados com MLflow autolog, incluindo métricas ROC-AUC, relatórios de classificação e confusion matrix. O modelo campeão é registrado no MLflow Model Registry com alias champion.

O notebook inclui EDA completa, insights sobre perfil de churn e implementação detalhada de todas as etapas do pipeline. Um threshold customizado de 0.35 foi definido para maximizar recall, priorizando a identificação correta de clientes em risco.
